# Product Sense

## Primary User

首版用户是“个人程序员型研究者”，不是机构量化团队。

## Product Promise

每天盘后，用户能稳定拿到三类结果：

1. 今天市场和个股发生了什么。
2. 今天哪些股票值得进入观察名单，以及为什么。
3. 这套规则在历史上表现如何，是否接近真实交易约束。

## Scope Discipline

首版优先级：

1. 数据链路稳定
2. 结果可解释
3. 回测可信
4. 自动化可运行

首版暂缓：

1. Tick/Level-2
2. 实盘交易
3. 复杂机器学习排序
4. AI 新闻总结型花活

## UX Rule

页面不只是“能看到数据”，还要让用户迅速回答：

1. 结论是什么
2. 依据是什么
3. 风险是什么
4. 下一步要不要看它
