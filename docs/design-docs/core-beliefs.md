# Core Beliefs

## Product Beliefs

1. 先把盘后研究闭环跑通，再扩分钟级和事件驱动。
2. 选股结果必须可解释，不能只给一个分数。
3. 回测真实性优先于漂亮收益率。
4. 单机可维护性优先于过早分布式化。

## Engineering Beliefs

1. 仓库里的事实优先于聊天里的共识。
2. boring tech 优先于黑盒魔法。
3. 边界处校验，内部尽量消费明确结构。
4. 快照和任务日志是系统契约，不是实现细节。
5. 能机械检查的约束，尽量不要依赖人工记忆。

## Agent Beliefs

1. 智能体需要地图，不需要一份失控膨胀的说明书。
2. 文档入口应稳定、短小、可交叉跳转。
3. 大任务要沉淀为执行计划，而不是一次性长提示。
4. 结构化约束会放大智能体吞吐，不会拖慢它。
5. 当智能体重复犯错时，应该优先修系统而不是加情绪化提示。
